Troubleshooting of FAST Search Configuration

FAST search as we all know is one of the nicest things to work with. Here I’m not going to describe what FAST search is or how to install and configure it, I did it few days back and the problem  faced actually started after the installation is done. But I was victorious at the end :).

*Note: During configuration see the different parameters in theD:/FASTSearch/ Install_Info.txt, you should use this during configuring FAST Search SSA or Query applications in Central Admin.
 
FAST Configuration:
 
The below article explains the configuration part very nicely, you should make sure that you follow each step as mentioned there. I liked this one.
http://community.bamboosolutions.com/blogs/sharepoint-2010/archive/2011/08/02/sharepoint-2010-cookbook-configuring-fast-search-server-2010-for-sharepoint.aspx

 

 
Now after configuration there could be many problems start appearing in the event logs, possible resolutions are mentioned below.
 
Issue 1: Failed to connect to <SERVER NAME:URL> :9999 Failed to initialize session with document engine: Unable to resolve Contentdistributor

 

 

 

Solution:
1>     First Check the certificate validation ( possible you have used a self certified certificate which might have expired in a year)
Click Start and then type MMC in the Search box–>Add/Remove Snap-in–>Select Certificates–>Select Computer account–>Select Local computer–>Click OK
 
2>     If you see the search certificate expired, renew it as follows
·         Navigate to folder “\installer\scripts” and run the following script:
               .\ReplaceDefaultCertificate.ps1 –generateNewCertificate $true
·         Check the data is updated
3>     After that run following power shell in “SharePoint 2010 Server Management Shell”  which should be run as Admin
D:\FASTSearch\installer\scripts\securefastsearchconnector.ps1 -certPath “D:\FASTSearch\data\data_security\cert\FASTSearchCert.pfx” -ssaName “FAST Search SSA” -userName “EU\gbsaisemoss_btest“
*Note: put correct Search service application and domain name for the red marked parameters.
Output:
Issue 2:  Failed to connect to <SERVER NAME:URL> :9178 Failed to create session with content distributor. Unknown collection ‘MyCollectionTest’
 
Solution:
This happens when collection specified the SSA is not present in the FAST database. To check if your collection exist you can execute the following command on the FAST server (SharePoint 2010 Mgmt Shell)
Get-FASTSearchContentCollection -Name MyCollectionTest
Output should be like this:
If the collection does not exist or else you can create a new collection with the following command
New-FASTSearchContentCollection -Name MyCollectionTest
For me it was not present hence gave me an error. For quick fix, I changed the collection name back to sp. It’s the default collection and was already present.
Issue 3: Access Denied to the Default Content Access Account
 
Solution:
Access is denied. Verify that either the Default Content Access Account has access to this repository, or add a crawl rule to crawl this repository. If the repository being crawled is a SharePoint repository, verify that the account you are using has “Full Read” permissions on the SharePoint Web Application being crawled.
This happens when the content access account does not have “Full Read” permissions on the SharePoint web application being crawled. You can add a crawl rule to access the repository with specific account.
1. Go to New Crawl Rule -> add the Url
2. Check the radio button for Specify a different content access account
3. Enter the credentials for the account with which you want to crawl.
4. Click Ok and start the crawl.
Also make sure there is an entry for DisableLoopbackCheck in the registry. If it’s not present follow the below steps:
1. Go to command window and type regedit.exe
2. Navigate to HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Lsa
3. Right click on LSA and create a new 32bit DWORD value
4. Name it as “DisableLoopBackCheck” and modify its value to 1
5. Close the registry and crawl the content.
Issue 4: Error communicating to the server
 
Solution:
The crawler could not communicate with the server. Check that the server is available and that the firewall access is configured correctly. If the repository was temporarily unavailable, an incremental crawl will fix this error.
This happens when crawler is not able to connect to the server. Make sure server name is correct. Couple of steps to troubleshoot it
1. You should be able to ping the server from the server having crawl component. Make sure there is an entry for the server in the host file under c:\Windows\System32\drivers\etc folder.
Ping
2. You should be able to connect to the server using telnet command
Telnet
Issue 5: Failed to connect to <SERVER NAME:URL>:9178 couldn’t locate certificate: CA89C82790724B01602616F9F7FCE7D74B886A12
 
Solution:
                  i.   Open the MMC as mentioned in Issue 1
                 ii.   Go to the FastSearchCertificate
               iii.   Expand the Personal folder and then click the Certificates folder. Double-click the CA signed certificate.
               iv.   Open the Details tab and then click Thumbprint.
                 v.   Check if this matched with the above error GUID i.e.  A89C82790724B01602616F9F7FCE7D74B886A12
               vi.   If not update the certificate as below
D:\FASTSearch\installer\scripts\securefastsearchconnector.ps1 -certPath “D:\FASTSearch\data\data_security\cert\FASTSearchCert.pfx” -ssaName “FAST Search SSA” -userName “EU\gbsaisemoss_btest“
 
 
Issue 6: Search in not working in any site, getting message: The search request was unable to connect to the Search Service (FAST Search Server 2010 for SharePoint)
 
Solution:
This message appears when you search any item in any site and you made sure that none of the above issues are appearing. Follow the steps mentioned here:
