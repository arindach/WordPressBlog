How to create multiple User Profile Service Application in SharePoint 2010

Arindam C
5 years ago

 

You might have requirement to create multiple user profile service application for various reason, like for me the requirement was to create a different mysite for a special project which will run in its own web application and some of the user profile properties will come from a secure database. So I decided to create another UPSA in addition to the existing one.
Now the catch here is, if you are working in a SharePoint farm with a single server, then there is no point creating more than one UPSA. The reason is UPSA is dependent on the user profile synchronization service and only one instance of this service can be run in a server. So if you create multiple UPSA, you have to select which one you want to associate with this user profile synchronization service, both USPA can’t be run simultaneously.
OK, so let me describe our farm architecture, we have 3 server farm, I’ll name then as SPWFE01, SPWFE02 and SPAPP01.  From the name you can understand what their functions are.
Since I have 3 server in the farm so I can run 3 UPSA in this farm, each UPSA can be associated with the user profile synchronization service in a server.
However I needed two UPSA, so the mapping is like below:
UPSA Name
Server where user profile synchronization service will run
User Profile Service 01
SPAPP01
User Profile Service 02
SPWFE01
Note: If you are in an existing farm ,most probably the default user profile service is already there, we will not touch that. In my case User Profile Service 01 was the default one and it was already running in the SPAPP01.
So we are going to create User Profile Service 02.

 You know the basic steps, you can also go here :http://www.sharepointboost.com/blog/configuring-the-user-profile-service-ups-in-sharepoint-2010/ for a detailed step by step guide.

  • Go to Application Management and then manage service application and create a new User profile service application.
  • Give the name as User Profile Service 02
  • Enter other information as per your need.

 

·         Important: In the “Profile Synchronization Instance” select SPWFE01 where the associated user profile synchorinization service from this UPSA will run.

·         Now hit create, in my case I gave a specific mysite URL also here which is not shown above.
·         Once this new service application is created, you can see that it’s listed in the service application page- OK so far so good.
Now we have to run this new UPSA in the SPWFE01. Go the Manage Services in this server page in the Central Admin.
Change the Server to SPWFE01; you will see that the User profile synchronization service is not running in this server. Click on start

 

In the next page you will now see there’s two UPSA appearing in the dropdown. The “User Profile Service 01” is already running in SPAPP01, so I will choose “User Profile Service 02” and put password and click OK

You have to wait a few minutes now; you will see the status of the service is “Starting“. After some time it should change to Started.  Voila now have two different UPSA running in your farm.

Troubleshoot : I have tested above process few times and it worked always, but there could be some error appear always.

Error: User Profile synchronisation  service is stuck in ‘starting’ for long long time

Solution: Most probably two forefront identity management services are not running in SPWFE01, go to services and check that, if not running  do a IISREST, restart SharePoint Timer Job and then start this 2 services. Now go and start the synchronisation service again.

Error : SP2010: FIMSynchronizationService errors when running Profile Synchronization,Profile import from AD fails. You are able to successfully create Synchronization Connection, but when trying to run Full or Incremental Profile Synchronization, you are greeted with errors in Application Event Log such as this:

The management agent “MOSSAD-[SYNCHRONIZATION CONNECTION NAME]” failed on run profile “DS_DELTAIMPORT” because of connectivity issues.

Solution:

  1. Start FIM SSM (C:\Program Files\Microsoft Office Servers\14.0\Synchronization Service\UIShell\miisclient.exe)
  2. Go to Management Agents screen, right click on MOSSAD-[SYNCHRONIZATION CONNECTION NAME] and select Properties
  3. Go to Configure Directory Partitions and uncheck the directory partition that you didn’t define in SP2010
  4. Click OK to close Properties window
  5. Again, right click on MOSSAD-[SYNCHRONIZATION CONNECTION NAME] and select Configure Run Properties…
  6. On DS_FULLSYNC, DS_DELTASYNC, DS_FULLIMPORT, DS_DELTAIMPORT, delete step for the directory partition you removed on bullet 3. NOTE! You must only remove the step for the directory partition you removed earlier on bullet 3. I.e., if the removed directory partition was 3rd on the list, you should delete 3rd Step.
  7. Click OK
  8. Re-run profile synchronization from Central Admin
