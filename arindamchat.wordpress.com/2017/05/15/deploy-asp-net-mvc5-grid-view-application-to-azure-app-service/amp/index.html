Create a MVC Azure Web App

You probably know that Azure App Service is a platform-as-a-service (PaaS) offering of Microsoft Azure. You can create web and mobile apps for any platform or device using .NET, Node.js, PHP, Python, and Java. You can integrate your apps with SaaS solutions, connect with on-premises applications or use other Azure service like Azure SQL Server etc. etc.  You can check more in the MS Azure Website

Here I’m going to describe how to deploy an ASP.NET MVC application as Azure App Service and use an Azure SQL Server as back end.

I hope that you already have an Azure Subscription before going to next sections.

The ASP.NET MVC Application

As it’s not the purpose of this demo to show how to create a MVC application from scratch, so I used this nice article named “Beginners Guide for Creating GridView in ASP.NET MVC 5” created by Ehsan Sajjad, and published in CodeProject. This nice article has the source code and the SQL scripts to create SQL tables and insert data to that. What could have been better than that?

Now, before you start creating necessary things in Azure, my suggestion will be to test this application locally first, see the article, follow the steps, create the DB locally and test it once.  I did it and this is working fine in my local machine.

Understanding the Azure App Service

Now let’s talk a little about things necessary to publish this application to Azure.  There are different options, we can create our own Virtual Machines, install IIS, .NET framework, SQL server and then install the application code, and this is the approach we are used to. Here you have to control and manage everything by yourself, the application VMs and SQL VMs.

But we have a better option also available in Azure, that is called Azure Web Apps, here we will not create a VM or install any software but will use the Azure infra to deploy out code. We have to decide how powerful infra we need to Application and Azure VM and have to pay accordingly. We will be using this approach here.

Azure Web App has two main components, App Service Plan and Azure App Service.

Azure App Service is the application/ code which runs on an App Service Plan. You can think App Service Plan as the selection of hardware on which your App Service i.e. the application code will run.

You can add your own code and subsequent modifications on an App Service and based on the App Service plan chosen, Azure will decide where the App Service will be deployed and we don’t have to bother about that. This why this is sometimes called as server less deployment.

In Azure it’s very easy to scale up or scale out an App Service plan based on your App Service need, if you need more performance for your App Service you can choose a premium App Service plan at any point of time in future.

App Service will have the application URL. Below diagram loosely shows the architecture I’m going to use, the blob storage is not used in this demo though.

Note that other than SQL and Blob, I there is “deployment slot” and Source control. I’ll also describe them in the following articles.

Create the App Service

First thing we need is to create the App Service on Azure.  Log in to Azure portal https://portal.azure.com/  using your subscription details. Now click on the App Service icon.

When you click on the Add icon, you will see there are various App Service Templates available as below.

Chose a specific App Service, lets choose Asp.Net Starter Service for this demo.

Fill the App service with desired name; remember that this name has to be unique in the azurewebsites.net. Note that I have chosen a new Resource group also here; this is a good idea to create a new resource group and keep everything inside that.

Notice that The App Service already chose a new App Service Plan, this can be used  but we want more control over it and we will create our own App Service plan.

So click on Create New, this will open the New App Service Plan blade, give desired name, now we can choose difference pricing plan here, based on the infra you choose you need to pay, for now I will choose Free plan, this will have shared hardware and custom domain names can’t be used, but this will service our purpose for this demo. If you choose a more powerful plan, the processes I’m going to describe below will not change.

So Select the Free tier and click on OK

Now you can see the newly created App Service plan has been selected for our App. Click on Create, this will create the new Azure Web App.

The process of creating a new App Service will take a bit of time; you can see the progress in the notification area on the top right corner.

And when done, you can see the success.

Click on the message and this will take you to the details of our newly created App Service, you can see Azure already added some monitoring also which is great for troubleshooting.

Note that now we have a URL for out app which is http://ac-aspmvc.azurewebsites.net/ , if you click on the link you will see a blank website as below. But this shows that our new App Service is up and running.

Use Visual Studio to connect to Azure.

Since we have the code already in my local Visual Studio, I can use this great tool to connect and then deploy to Azure also.  Go to Visual Studio (note that I’m using Visual Studio 2015) and go to View –> Cloud Explorer

 

Note: If you don’t see the cloud explorer of even if you see the option makes sure to have the latest version of this , for that go to Toolsà Extension and Updates and check if the Cloud Explorer for Visual Studio is installed, make sure to check Automatically update. If this is not done, you might get error in next steps.

Since I have already connected to my Azure subscription, I can see the connection details here, for you enter Azure subscription details here to connect to Azure.

 

Once connection is done, you should be able to view the newly created App Service here.

Deploy the code to Azure

Now that your Visual studio is connected, let’s deploy the code to Azure App Service i.e. AC-ASPMVC. Right click on the solution and choose Publish

This will open the Publishing Wizard and you can see there is an option to deploy in Azure App Service. Click on that.

Now you can see that our App Service AC-ASPMVC is available, select that and click OK

Note: Here you can create a new App Service also to be deployed in the Azure, in that case a new App Service and App Service Plan will be created and then deployment will happen, but this is not needed here as we want to deploy on our existing App Service.

Now, you can see that it’s trying to connect to the Azure, the User name and password is important thing, but this will be discussed in more details in next article, for now click on Validate.

If validation is successful, click on next, here if your code is using any file service you can configure that here, but the most important thing is the connection sting, you can specify the DB connection string here, since we don’t have the Azure DB yet, we will leave it as is. Click on Publish

You can see the build has been started and you can see the output.

If deployment is successful, this will open our App Service URL i.e. http://ac-aspmvc.azurewebsites.net/  and you can see a big error here.

This is fine, as we don’t have the DB configured yet, so we have this error, but the deployment  worked here.

Create Azure SQL Database.

This application is using a database, so let’s create an azure SQL database. Now when we create a database in Azure, along with the database name, a ‘server’ name is also required.  This ‘server’ is not a physical or even a virtual server, this is basically a logical container name which can hold multiple databases, and where the actual databases are stored in Azure is not our concern, because we can use the logical container or logical server name to connect to any of the database it’s holding and this server name can also be used in a connection string.

So from the Azure menu, click on SQL Databases.

The application I have downloaded from code project is using the database name as “GridExampleMVC”, so I’ll keep the same DB name here also; I’ll not create a new resource group, but will use the existing one. Now you can see that you can either create a blank DB or restore from a backup of create a sample DB. Let’s choose blank DB.

SQL elastic pool is not required here, as we are just going to use it for demo purpose, though practically other than huge Db, this option is not required as the associated cost is too high.

Now click on the arrow of Server-Configure required settings. This will let us know the logical server name.

Fill the details, you can choose your desired user ID here, remember this will be SQL Admin login for all your SQL Dbs under this server. There’s no option for windows authentication in Azure DB.

On creation, you can see the server ‘ac-aspmvs-sql’ is now selected. Now click on the Pricing Tier arrow, here you can choose different options, for now select the Basic

The configuration looks like below, now click on the Create button.

Again this will take some time to be created, you can check in the notification area and also verify that deployment is done.

Now go back to the newly created DB details. This looks like below, and the interesting thing is here you can see the DB name and connection details.

Click on Show Database Connection String and you can see the connection below.

Note that this is not the working connection string; you need to fill the user name and password here. But we will do this latter.

Connect to Azure Database via sql management studio.

Now we need to create the tables and insert some records, we can use SQL Management Studio to connect to Azure, but before that we need to modify the firewall rule for the azure database. Azure DB won’t allow connection otherwise. To do that click on the Set Server Firewall

You need to add client IP in the firewall, you can see the client IP already there. Click on the Add client IP, this will add the client IP in the rule. Don’t forget to Save the change.

From the Azure portal we know that the server name is “ac-aspmvc-sql.database.windows.net”, so try to connect to that as Server, remember that we created the SQL Admin user id and password beforehand, use that. Now you will have the Azure DB opened.

 Connect to Azure Database using azure tool.

We can also use the Azure portal tool, this is still in preview, but I want to show you the usage, we will upload a SQL script to create a table and insert data. Open the Azure SQL database blade and you can see tools on the top ,click on that.

Choose Query Editor

It will ask for authentication, use the same SQL Admin user ID and password.

I have already saved the DB related scripts from Code Project; we will now upload this to run.

Click on the Open Query and select the .sql script

Once this is loaded, click on RUN, the script will start executing and you can see the output that one table has been created and 1000 rows are inserted.

Let’s check the DB with a small query, this you can type there. The below query will show the 100 records from Azure DB

Select top 100* from Assets

Ok, so our DB is now ready to be used.

Use Connection STRING TO the Application

So in previous steps we have seen the DB connection string in the portal, let’s copy that, you can see that we need to pass the SQL user name and password. For here I’ll use the admin username i.e. arindach and the password, but in practical scenario you should create some other user and this Admin user should not be used with others.

Server=tcp:ac-aspmvc-sql.database.windows.net,1433;Initial Catalog=GridExampleMVC; Persist Security Info=False; User ID={your_username}; Password={your_password}; MultipleActiveResultSets=False;Encrypt=True;TrustServerCertificate=False;Connection Timeout=30;

Now to add this connection string to Azure, go to the App Service and open the details blade of our Application.  You can see there is an Application settings option in the left menu, open this and scroll down.  There you can see a section for Connection Settings; we need to add the ‘DefaultConnection’ (as in the web.config of our application). Make sure to check the slot settings, this will ensure that when you publish this configuration will not be overridden by your local connection string.

Check the application

Now that we have DB connection string in place and we have a DB with all the records, let’s refresh the web browser with the Azure application URL i.e. http://ac-aspmvc.azurewebsites.net/  . And you can see that my application is up and running now.
