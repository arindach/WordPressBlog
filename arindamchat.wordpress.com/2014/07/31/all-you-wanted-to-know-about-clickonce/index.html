All you wanted to know about ClickOnce

What is ClickOnce?

MS says: ”ClickOnce deployment allows you to publish Windows-based applications (and WPF also) to a Web server or network file share for simplified installation. Visual Studio provides full support for publishing and updating applications deployed with ClickOnce. ClickOnce deployment is available for projects created with Visual Basic, Visual C#, and Visual J#, but not for Visual C++.”

So you developed a windows application and now want to distribute it over Internet and also don’t want to inform each users when a new version is released in future, you need ClickOnce.

Your end user can download the Clickonce from a URL and whenever they start the app, it will check if there’s a new version released in the deployment server, if it is the new version will be downloaded automatically.

Check here for more: http://msdn.microsoft.com/en-us/library/aa480721.aspx
If you are totally new you can check here: Step-by-Step: A First ClickOnce Application

Publishing ClickOnce:

In your Dev Box:

Now you can publish a Windows/WPF application many way, DiskBased, a network shared location or ftp or in web. For ClickOnce you should use the Web deployment only.

The ideas is clear, you are done with your Windows Application development. Then in your Dev box you just give a localhost url and publish, a new website will be created with some files.

If you check that step by step article, in step 3 you give a URL, if you already have a WebSite you can give that path, otherwise the publishing process will create the folder in C:\inetpub\wwwroot and a new website will be added to IIS.

For now think that I published it to my Dev IIS to: http://localhost/WinFrom1Local , if this website is not in IIS already, a folder will be created as C:\inetpub\wwwroot\WinFrom1Local and the website will also be create automatically. Below is a sample screenshot.

What Happens when you Publish:

You must have seen the file structure once you publish your application to a URL, below screenshot shows the structure. This is the virtual folder of my Dev IIS website where I published the app (C:\inetpub\wwwroot\WinFrom1Local).





Application Files folder: This contains the different version of app files, each time you publish a new version will be created here. When end users open application, the app will check for any new version here and download if required.

Setup.exe: This has the information from where rest of the required files to be downloaded. This download location will be the Publish URL you gave during Publish, in our case it will be http://localhost/WinFrom1Local.

This you can see, open the setup.exe in notepad, search for “”, you will see like below, the localhost has been replaced with the actual machine name.

 

What about in my Production Box

The above technique will work in your Dev box; you can’t publish your application to any remote IIS hosted web site, mean from your Dev box you can’t directly publish to your Production site URL.

Also if you publish your app in Dev box to http://localhost/WinFrom1Local and then copy the output from path “C:\inetpub\wwwroot\WinFrom1Local” and put in the virtual directory of your Production box, the Publish.htm will open, but when you click on Install you will get some error like this

My Experience:

This is a good technology. I was able to make it working in all scenarios, except with SSO. The situation was, you need to deploy the App to a website in a server secured by SSO, mean whatever URL you try to open it will first open the SSO page and after successful login it will redirect to the actual page. In this scenario, it always failed to download .manifest or other files, I have seen the error log and it appeared that even after I do a successful login to SSO page, when I click on the Install button, it tries to authenticate again with the SSO server and fails. Most probably reason is ClickOnce engine does not support cookie. If you see the above article for Form Based Authentication, this is the reason you need to do a cookie less authentication.
