SharePoint Library with Tree View

Arindam C
3 years ago
Recently I was working in a project where the requirement was to show a tree view structure along with a SharePoint 2010 document library.
Let’s describe the actual requirements:
·         In this project whenever a record will be generated a new folder with record ID will be created inside document library.
·         Inside the Record Number folder a predefined folder structure will also be created.
·         Users can add new folder to this structure but can’t delete the original structure.
·         The folder structure will be shown as a tree-view left side and users can do normal SharePoint library operations.

 

Above diagram shows the look and feel. A probable way could have been using managed metadata as it gives you the tree view structure OOB. But here we need to show normal folders in the document library.

So I decided to create a new view and modify that to add the tree view.
The important question was how to create a tree-view, there is nothing out of box here. So we need to create a custom control for this. Fortunately we were using Telerik control in the project and Telerik has an excellent tree view control. It’s called RadTreeView, you can check it here: http://demos.telerik.com/aspnet-ajax/treeview/examples/overview/defaultcs.aspx
Now in this control may not be available to you. So you have to create a custom control of your own.
But the good thing here is I’ll show how you can generate a XML from SharePoint library which can be readily used with Telerik control as well as you create any custom control.
Create XML for Tree-View from SharePoint folder library structure.
I wrote a recursive method to loop through the folder structure and create a XML. The code is shown below.
LoadTreeView: In this method I’m passing the record Id which will be the root folder. Then I’m calling CreateFolderXML() which is the recursive function.
protected void LoadTreeView(string recordId)
        {
            SPSecurity.RunWithElevatedPrivileges(delegate()
            {
                SPFolder folder = SPContext.Current.Web.GetFolder(SPContext.Current.Web.ServerRelativeUrl + new CaseDocumentRepository(Convert.ToInt32(caseId)).GetListName() + “/” + recordId);
                varxmlRoot = new XElement(“Tree”, DocumentTreeViewService.CreateFolderXML(folder, recordId));
                var doc = newXDocument(newXDeclaration(“1.0”, “UTF-8”, “yes”), xmlRoot);
                RadTreeView1.LoadXml(doc.ToString());
            });
        }
CreateFolderXML: This is the recursive method which takes the root folder and loop through, it adds directory name and the tree-view node text and a folder image to show the yellow folder and then the URL for clicking.
One important thing I want to note here, when you have a lots of folder in your library (in my case I have 70000 record ID as root folder so you can think about the load), this type of recursive call can take long time. So I used LINQ to query the SP folders instead on conventional CAML query or normal GetFolder as below
List<SPFolder> folderList = dir.SubFolders.Cast<SPFolder>()
                .Where(p => p.Name.ToLower() != “_w”&& p.Name.ToLower() != “_t”)
                .OrderBy(li => li.Name.Trim()).ToList();
Where(p => p.Name.ToLower() != “_w” && p.Name.ToLower() != “_t”)- this is used because you can use the same code on a picture library and exclude smaller thumbnalis which stored under _w or in _t folder ( when you uploads a picture in Picture library SharePoint create 3 copies of it)
public static XElement CreateFolderXML(SPFolderdir, string caseID)
        {
            XAttribute[] content;
            XElement xmlInfo;
            if (dir.Name == caseID)
            {
                content = new XAttribute[4];
                content[0] = new XAttribute(“Text”, dir.Name);
                content[1] = new XAttribute(“Value”, “/” + dir.Url + “&View=” + dir.DocumentLibrary.DefaultView.ID);
                content[2] = new XAttribute(“ImageUrl”, “/_layouts/images/folder.gif”);
                content[3] = new XAttribute(“Expanded”, “True”);
            }
            else
            {
                content = new XAttribute[3];
                content[0] = new XAttribute(“Text”, dir.Name);
                content[1] = newXAttribute(“Value”, “/” + dir.Url + “&View=” + dir.DocumentLibrary.DefaultView.ID);
                content[2] = new XAttribute(“ImageUrl”, “/_layouts/images/folder.gif”);
            }
            xmlInfo = new XElement(“Node”, content);
           
            List<SPFolder> folderList = dir.SubFolders.Cast<SPFolder>()
                .Where(p => p.Name.ToLower() != “_w”&& p.Name.ToLower() != “_t”)
                .OrderBy(li => li.Name.Trim()).ToList();
           
            foreach (SPFoldersubDir in folderList)
            {
                xmlInfo.Add(CreateFolderXML(subDir, caseID));
            }
            return xmlInfo;
        }
Create a new Custom View
You can modify default allitems.aspx, but I would say you create a new view and keep the default one. You can just use SharePoint designer to create a new view.
Create Webpart for the TreeView
Just create a new Visual webpart for RadTreeview, you can placed your own custom control here if you don’t use Telerik.
<script language=”javascript” type=”text/javascript”>
      function selectNode(text) {
          var treeView = $find(“<%= RadTreeView1.ClientID %>“);
          var node = treeView.findNodeByText(text);
          node.select();
      }

 

</script>
