Different App-Hosting Option in SharePoint 2013

Arindam C
4 years ago
I spent some time to obtain knowledge about creating an app-hosting environment for developers. I fell this is a little confusing with all different options you have for your app-hosting. Obviously MSDN sites helped. You can use either an Office 365 Developer site to host your apps of can use a On premise SharePoint 2013 development machine for developing and testing apps locally, there is pros and cons.
Below I copied some relevant text from MSDN article:
Determine the right development environment for the type of app for SharePoint you’ll create
Before you begin creating and publishing apps for SharePoint, think about the best place to develop them—on an Office 365 Developer Site or on a local computer where you have SharePoint 2013 installed. Consider how you want to use the apps, the time and money you have for development, and the permissions you want for the apps.
Ø  An Office 365 Developer Site is preconfigured for app isolation and OAuth, so this option saves you the most time and expense, and enables you to develop and test apps for SharePoint on any computer and operating system on which you can install Visual Studio 2012. An Office 365 Developer Site is also the best option if you want to deploy directly from Visual Studio 2012 when you are developing and testing your apps for SharePoint.
Ø  If you want to distribute apps for SharePoint through an app catalog, such as apps for SharePoint that have full control permissions (which you cannot deploy to an Office 365 site), you’ll want to work with a full installation of SharePoint 2013 on a local computer.
