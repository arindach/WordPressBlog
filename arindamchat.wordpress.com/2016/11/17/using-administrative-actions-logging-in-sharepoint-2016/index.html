Using Administrative Actions logging in SharePoint 2016

So Microsoft has released the first SharePoint 2016 Feature Pack 1. Now if you wonder what a feature pack is you can check this blog. The exact words are as below:

Unlike previous versions of SharePoint, release-to-manufacture (RTM) did not define the end of innovation, but the beginning. As we continue to develop SharePoint Server 2016, we’ve paid close attention to customer feedback, trends in content management, team collaboration, user experiences across devices, and how the cloud can be blended into existing on-premises scenarios in new and compelling ways. Feature Packs allow us to accelerate delivery of cloud-first features to our Software Assurance customers (where noted) of SharePoint Server 2016 outside of the traditional 2- to 3-year release cadence

It seems to me that instead of publishing big software patches Microsoft probably trying to publish smaller patches at a time.

This pack consists of many features like MinRole enhancements, some more features for OneDrive for Business experience, Hybrid capabilities, OneDrive API 2.0 etc. but here I’ll discuss “Administrative actions logging”

Now even Admins are under surveillance, all their activities can be tracked and logged.

There is already a nice article in TechNet, please check the article. I don’t want to repeat the steps already described there but the interesting point according to me is now you can query to SharePoint DB directly to view admin activity logs. Off course the old way of checking ULS logs and using PowerShell to merge ULS logs are available.

Here I’m showing a high level steps to configure and view Admin activity logs.

 Configure

You can configure “Administrative Actions” in central admin (You can do so using PowerShell also)

Find the Administrative actions local log file location

From Central Admin –>Monitoring –>Configure usage and health data collection –>Usage Data Collection Settings- you can find the ULS log file location.

Find the Administrative actions Usage Database log files

This is the new thing I was talking about. From Central Adminà Monitoring–>Configure usage and health data collection–> Logging Database Server settings– here you can see the DB where the logs are storing.

Retrieving logs from the SharePoint Usage Database

The TechNet article already have queries, you can use then directly or can modify as per your need. Below screenshot shows the SQL DB and output. You can see the correlation ID also in the DB, so you can search in ULS for more information if the Details column is not giving you sufficient information to you.
